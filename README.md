# Тестовое задание ShipIt

## Описание

ShipIt это небольшое приложение для оформления заказов на доставку.

Приложение состоит из ASP.NET Core backend и Blazor frontend. Данные хранятся в SQLite, отдельный сервер базы данных не требуется.

## Функционал

### Создание заказа

Форма содержит следующие поля:

- город отправителя;
- улица и дом отправителя;
- квартира отправителя, если есть;
- город получателя;
- улица и дом получателя;
- квартира получателя, если есть;
- вес груза;
- дата получения груза.

После сохранения заказу автоматически присваивается номер. Затем открывается страница с информацией о созданном заказе.

Все поля обязательны, кроме квартиры отправителя и квартиры получателя.

### Список заказов

На странице /orders отображаются все созданные заказы. Новые заказы находятся в начале списка. Нажатие на строку открывает подробную информацию.

### Информация о заказе

Страница /orders/{id} показывает данные выбранного заказа без возможности редактирования.

## Структура проекта

- backend содержит API, бизнес-логику, работу с SQLite и автоматические тесты;
- frontend/ShipIt.Client содержит Blazor WebAssembly приложение;
- README.md содержит команды запуска и порядок ручной проверки.

## Основные решения

- ASP.NET Core 9 используется для backend.
- Blazor WebAssembly на .NET 9 используется для frontend.
- SQLite позволяет запустить проект без дополнительной настройки базы данных.
- Миграции применяются автоматически при запуске backend в режиме разработки.
- Если база пустая, backend добавляет тестовые заказы.
- Проверка формы выполняется через DataAnnotations. Backend повторно проверяет входные данные перед сохранением.
- Адрес frontend для CORS ограничен значением http://localhost:5026.

## Что нужно для запуска

- .NET 9 SDK
- современный браузер

## Запуск через терминал

В корневой папке проекта запустите backend:

dotnet run --project backend/src/ShipIt.Api/ShipIt.Api.csproj

В отдельном терминале запустите frontend:

dotnet run --project frontend/ShipIt.Client/ShipIt.Client.csproj --launch-profile http

После запуска доступны:

- frontend: http://localhost:5026
- API: http://localhost:5100
- Swagger: http://localhost:5100/swagger

## Запуск через IDE

1. Откройте решение backend/ShipIt.sln.
2. Запустите проект backend/src/ShipIt.Api/ShipIt.Api.csproj.
3. Запустите проект frontend/ShipIt.Client/ShipIt.Client.csproj с профилем http.
4. Откройте http://localhost:5026.

## Тесты

В корневой папке проекта выполните:

dotnet test backend/ShipIt.sln

## Ручная проверка

1. Запустите backend и откройте http://localhost:5100/swagger.
2. Запустите frontend и откройте http://localhost:5026.
3. Убедитесь, что на странице /orders отображаются тестовые заказы.
4. Нажмите на заказ и проверьте страницу с информацией о нём.
5. Откройте /orders/create, заполните все поля и отправьте форму.
6. Проверьте, что открылась страница нового заказа с автоматически созданным номером.
7. Вернитесь на /orders и убедитесь, что новый заказ появился в списке.
