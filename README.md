# Тестовое задание ShipIt

## Описание

ShipIt это небольшое приложение для оформления заказов на доставку.

Приложение состоит из ASP.NET Core backend и Blazor frontend. Данные хранятся в SQLite, отдельный сервер базы данных не требуется.

## Функционал

### Создание заказа

Форма содержит следующие поля:

- город отправителя;
- улица и дом отправителя;
- квартира отправителя, если есть;
- город получателя;
- улица и дом получателя;
- квартира получателя, если есть;
- вес груза;
- дата получения груза.

После сохранения заказу автоматически присваивается пользовательский номер вида ORD-20260704-ABC123. Также у заказа есть технический Id, который используется API для открытия деталей заказа.

Все поля обязательны, кроме квартиры отправителя и квартиры получателя.

### Список заказов

На странице /orders отображаются созданные заказы постранично. Новые заказы находятся в начале списка. Нажатие на строку открывает подробную информацию.

### Информация о заказе

Страница /orders/{id} показывает данные выбранного заказа без возможности редактирования.

## Структура проекта

- backend содержит API, бизнес-логику, работу с SQLite и автоматические тесты;
- frontend/ShipIt.Client содержит Blazor WebAssembly приложение;
- README.md содержит команды запуска и порядок ручной проверки.

## Основные решения

- ASP.NET Core 9 используется для backend.
- Blazor WebAssembly на .NET 9 используется для frontend.
- SQLite позволяет запустить проект без дополнительной настройки базы данных.
- Миграции применяются автоматически при запуске backend в режиме разработки.
- Если база пустая, backend добавляет тестовые заказы.
- Проверка формы выполняется через DataAnnotations. Backend повторно проверяет входные данные перед сохранением.
- API возвращает отдельные response DTO и не раскрывает domain/EF модели.
- GET /api/orders поддерживает пагинацию через pageNumber и pageSize.
- Непредвиденные ошибки обрабатываются глобально и возвращаются в формате ProblemDetails без внутренних деталей исключения.
- Backend прокидывает CancellationToken из HTTP запроса в validation, handlers, repository и EF Core операции.
- Адрес frontend для CORS ограничен значением http://localhost:5026.

## Идентификаторы заказа

- Id это технический идентификатор для API, базы данных и маршрутов вида /orders/{id}.
- OrderNumber это пользовательский номер заказа, который отображается в интерфейсе.
- Формат номера: ORD-yyyyMMdd-XXXXXX, например ORD-20260705-A1B2C3.
- ORD это префикс заказа, yyyyMMdd это дата, XXXXXX это короткая случайная часть.
- OrderNumber уникален и удобнее GUID для диктовки, печати в документах и общения с поддержкой.

## API

### Создать заказ

POST /api/orders

Успешный ответ содержит технический id и пользовательский orderNumber.

### Получить список заказов

GET /api/orders?pageNumber=1&pageSize=20

Параметры:

- pageNumber по умолчанию 1;
- pageSize по умолчанию 20;
- максимальный pageSize равен 100.
- pageNumber должен быть больше или равен 1;
- pageSize должен быть от 1 до 100;
- заказы возвращаются от новых к старым по CreatedAtUtc.

Ответ:

{
  "items": [],
  "pageNumber": 1,
  "pageSize": 20,
  "totalCount": 0,
  "totalPages": 0
}

Если pageNumber или pageSize переданы некорректно, API возвращает validation ProblemDetails с кодом 400.

### Получить заказ по Id

GET /api/orders/{id}

id это технический идентификатор заказа. Пользовательский номер заказа возвращается в поле orderNumber.

## Обработка ошибок

Ожидаемые ошибки, например ошибки валидации, отсутствие заказа или конфликт номера заказа, возвращаются через ProblemDetails с подходящим HTTP статусом.

Непредвиденные ошибки проходят через ExceptionHandlingMiddleware. Middleware логирует ошибку и возвращает безопасный ProblemDetails со статусом 500, без stack trace и внутренних деталей исключения.

Если клиент отменил запрос, OperationCanceledException не считается ошибкой сервера.

## Что нужно для запуска

- .NET 9 SDK
- современный браузер

## Запуск через терминал

В корневой папке проекта запустите backend:

dotnet run --project backend/src/ShipIt.Api/ShipIt.Api.csproj

В отдельном терминале запустите frontend:

dotnet run --project frontend/ShipIt.Client/ShipIt.Client.csproj --launch-profile http

После запуска доступны:

- frontend: http://localhost:5026
- API: http://localhost:5100
- Swagger: http://localhost:5100/swagger

## Запуск через IDE

1. Откройте решение backend/ShipIt.sln.
2. Запустите проект backend/src/ShipIt.Api/ShipIt.Api.csproj.
3. Запустите проект frontend/ShipIt.Client/ShipIt.Client.csproj с профилем http.
4. Откройте http://localhost:5026.

## Тесты

В корневой папке проекта выполните:

dotnet test backend/ShipIt.sln

## Ручная проверка

1. Запустите backend и откройте http://localhost:5100/swagger.
2. Запустите frontend и откройте http://localhost:5026.
3. Убедитесь, что на странице /orders отображаются тестовые заказы и доступны кнопки переключения страниц.
4. Через Swagger вызовите GET /api/orders?pageNumber=1&pageSize=20 и проверьте поля items, pageNumber, pageSize, totalCount, totalPages.
5. Через Swagger вызовите GET /api/orders?pageNumber=0&pageSize=20 и проверьте, что вернулся validation ProblemDetails.
6. Нажмите на заказ и проверьте страницу с информацией о нём.
7. Откройте /orders/create, заполните все поля и отправьте форму.
8. Проверьте, что открылась страница нового заказа с автоматически созданным пользовательским номером.
9. Вернитесь на /orders и убедитесь, что новый заказ появился в списке первым.
